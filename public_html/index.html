Deploying simple websites with Git

This fine website and the blog contained herein are entirely maintained and deployed with git. I edit a local repository on my laptop and commit to my linode, where a post-receive hook pulls down the changes to the live version of my site.

Obviously there are some better solutions for web applications to handle all the moving pieces (e.g. Capistrano) but for simple sites I find this workflow quite pleasant.

To get started you need to create a bare Git repository on your server. Further examples will pretend this repo lives in /var/git so that may be a good place for it. I will also assume that you have excellent taste and have therefore chosen to name your repository 1337807.git.

git --bare init

Now we're going to create the local repository for your site and add the remote repository as origin.

git init
git add remote origin your_user@your_server:/var/git/1337807.git
